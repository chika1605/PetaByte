# Axelor Demo Project — PetaByte

Демо‑проект на базе **Axelor Open Platform**, включающий кастомный модуль управления задачами `demo-task` с собственными моделями, представлениями и бизнес‑логикой.

Приложение написано на Java и использует базу данных PostgreSQL, поднятую в Docker.

---

## Описание проекта

Проект предназначен для демонстрации:
- создания собственного модуля Axelor;
- описания сущностей и представлений;
- реализации простой бизнес‑логики (управление задачами).

Основной модуль: `demo-task`.

---

## Требования

- установленный **Docker Desktop**;
- установленная **JDK 11+** (рекомендуется LTS);
- доступ в интернет для скачивания зависимостей Gradle (используется wrapper `gradlew`).

---

## Как запустить проект локально

Запуск состоит из двух шагов: сначала поднимаем базу данных PostgreSQL в Docker, затем запускаем само приложение через Gradle.

### Шаг 1. Запуск базы данных (PostgreSQL)

База данных работает в Docker‑контейнере. В конфигурации (`axelor-config`) порт БД изменён на `5433`, чтобы избежать конфликтов с локально установленным PostgreSQL.

1. Убедитесь, что **Docker Desktop** установлен и запущен.
2. Откройте терминал в корневой папке проекта `PetaByte`.
3. Запустите контейнер с базой данных:

   ```bash
   docker-compose up -d
   ```

   Флаг `-d` означает *detached mode* — контейнер продолжит работать в фоне.

### Шаг 2. Запуск приложения Axelor

Для сборки и запуска используется Gradle Wrapper: `gradlew` (Windows) или `./gradlew` (Linux/MacOS).

1. В том же терминале выполните:

   ```powershell
   .\gradlew clean run
   ```

   При повторных запусках можно использовать более короткую команду:

   ```powershell
   .\gradlew run
   ```

2. Дождитесь сообщения об успешном запуске Tomcat (в конце логов будет строка вида `Tomcat started on port(s): 8080`).

## Как открыть приложение

После успешного старта приложения откройте браузер и перейдите по адресу:

`http://localhost:8080/`

По умолчанию (для демо‑настроек Axelor) используются следующие учётные данные:
- **Username**: `admin`
- **Password**: `admin`

Либо используйте креденшалы, указанные в ваших конфигурационных файлах.

## Структура и функциональность модуля `demo-task`

Модуль `demo-task` расположен в директории `modules` и содержит:
- сущность `Task` (название, приоритет, дедлайн и др. поля);
- табличное и форменное представления (например, `Task.xml`);
- бизнес‑логику в `TaskController.java`, включая:
  - автоматическую установку статуса `Overdue` при просрочке дедлайна;
  - дополнительные проверки и обработку действий пользователя;
- визуальную индикацию строк в списке задач (цвета в зависимости от статуса/дедлайна);
- быстрые фильтры (например, «Мои задачи», «Завершённые задачи» и др.).

## Полезные команды

- остановить базу данных:

  ```bash
  docker-compose down
  ```

- посмотреть логи базы данных:

  ```bash
  docker logs axelor-demo-postgres
  ```

- остановить приложение:

  в терминале, где запущен Gradle, нажмите `Ctrl + C` и подтвердите завершение.
